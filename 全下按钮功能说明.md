# 全下按钮二次确认功能

## 功能说明

玩家点击"全下"按钮时，弹出确认对话框，防止误操作。

### 对话框内容
- **标题**: "我要发狂了！！！"
- **按钮1**: "点错了" - 取消操作，无事发生
- **按钮2**: "发狂了" - 确认执行全下

### 实现位置

**组件**: `client/src/components/ActionBar.jsx`

主要改动：
1. 添加状态 `showAllInConfirm` 控制对话框显示
2. 原 `handleAllIn` 改为显示对话框
3. 新增 `confirmAllIn` 执行全下操作
4. 新增 `cancelAllIn` 取消操作

**样式**: `client/src/styles/actions.css`

新增样式：
- `.allin-confirm-overlay` - 半透明遮罩
- `.allin-confirm-modal` - 对话框主体
- `.allin-confirm-text` - 标题文字
- `.allin-btn-cancel` - 取消按钮
- `.allin-btn-confirm` - 确认按钮

## 测试步骤

### 手动测试
1. 启动服务器和客户端
2. 进入房间，开始游戏
3. 等待轮到你操作
4. 点击"全下 XXX"按钮
5. **验证**: 弹出确认对话框，显示"我要发狂了！！！"
6. 点击"点错了"
7. **验证**: 对话框关闭，未执行全下操作
8. 再次点击"全下 XXX"按钮
9. 点击"发狂了"
10. **验证**: 对话框关闭，执行全下操作，其他玩家看到你的全下行动

### 检查清单
- [ ] 对话框正确显示

- [ ] "发狂了"按钮执行全下
- [ ] 对话框样式美观（红色警告风格）
- [ ] 点击遮罩层外区域不关闭对话框
- [ ] 对话框有动画效果（淡入 + 上滑）

## 效果演示

```
玩家点击: [全下 1000]
    ↓
弹出对话框:
┌─────────────────────────────┐
│                             │
│   我要发狂了！！！          │
│                             │
│   [点错了]  [发狂了]        │
│                             │
└─────────────────────────────┘
    ↓
点击"发狂了" → 执行全下
点击"点错了" → 无事发生
```

## 注意事项

- 对话框使用 `position: fixed`，始终居中显示
- 遮罩层背景 `rgba(0, 0, 0, 0.7)`，半透明黑色
- 对话框边框使用红色 `var(--danger)`，强调警告
- 动画时长: 淡入 0.2s，上滑 0.3s
- 按钮悬停效果: 上移 2px + 阴影变化
